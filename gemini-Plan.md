# 제5회 인공지능 모델링 경진대회 - Gemini CLI 전략 분석 및 추천 로드맵

이 문서는 **Gemini CLI**가 분석한 현재 코드베이스의 강점/한계와 실시간 실증 실험 결과를 바탕으로, 소스 개발 담당인 **Claude**가 즉시 구현하고 실험할 수 있도록 정리한 전략 및 개발 가이드라인입니다.

---

## 1. 현재 코드베이스 종합 평가

### 🟢 핵심 강점 (Best Practices) - **1위 탈환 성공**
* **전략 28 (PL2 + Unit Price 8-model Ridge)의 성공**: 
    * **Pseudo Label v2 (PL2)**: 모델 간 합의도와 Fold 간 변동성을 이용한 데이터 증강으로 Public 점수를 크게 개선.
    * **평당가(Unit Price) 모델의 다양성**: `Target/Area`를 타겟으로 하는 모델이 단독 성능은 낮지만, 기존 `log/raw` 모델들과 결합했을 때 유일하게 유효한 **다양성(Diversity) 축**으로 작용하여 Ridge 스태킹의 성능을 극대화함.
* **단계별 레이어 파이프라인 (L1 ~ L4)**: 체계적인 구조 덕분에 새로운 모델(Unit Price 등)을 레이어에 자연스럽게 통합하고 실험할 수 있음.
* **구별 트렌드 보정 (GTR)**: 시간적 단절을 극복하는 핵심 치트키로 여전히 유효함.

### 🔴 해결해야 할 과제 (Remaining Challenges)
* **고가/대형 아파트 과소 예측 (High-end Underprediction)**: 
    * Error Analysis 결과, 120㎡ 이상 대형 평수와 강남/성동/용산 등의 고가 지역에서 오차가 집중됨.
    * 이는 `log` 스케일 모델의 특성과 `raw` 스케일 모델의 한계가 복합적으로 작용하여 발생하는 편향(Bias)임.
* **OOF와 Public의 괴리**: 전략 28에서 보듯 OOF보다 Public 점수가 훨씬 좋게 나오는 현상이 발생하므로, 단순 OOF 개선보다는 리더보드 패턴(Distribution shift)에 강건한 모델링이 필요함.

## 2. 실증 실험 데이터 및 인사이트

### 💡 핵심 인사이트: "다양성이 곧 성능이다"
* **실패한 시도들**: 
    * 타겟 변환 (`sqrt`, `Target/Floor`, `Target/Area*Floor`) $ightarrow$ 패턴이 유사하거나 오차를 증폭시켜 다양성 기여 실패.
    * 피처 변형 (상대 피처, TE 추가) $ightarrow$ 트리 모델의 자동 학습 및 정보 중복으로 효과 미미.
    * 반복 PL (Round 2+) $ightarrow$ 편향(Bias)이 강화되어 오히려 성능 악화.
* **성공한 시도**:
    * **평당가(Target/Area) 모델**: 기존 모델들과 예측 분포가 달라 Ridge 스태킹 시 강력한 보완 효과를 제공함.

## 3. Claude를 위한 단계별 개발 로드맵 (Immediate Action Plan)

현재 1위 전략(전략 28)을 베이스라인으로 삼아, 성능을 굳히거나 추가 돌파를 위한 단계별 태스크입니다.

### 🏃 태스크 1. [고도화] Multi-Dimension Scale-Blending (8모델 $ightarrow$ 12모델+)
* **목표**: `Unit Price` 외에 다른 관점의 모델을 추가하여 Ridge 스태킹의 다양성을 극대화.
* **Claude 구현 가이드라인**:
    1. 기존 8모델 (CB/LGB $	imes$ log/raw + Unit Price 4모델) 유지.
    2. **신규 모델 후보군 탐색 (실험적 접근)**: 
       * `Area` 기반의 다른 스케일링 혹은 매우 약한 규제가 적용된 모델 등, 기존 8모델과 **상관관계(Correlation)가 낮은** 모델을 1~2개 추가하여 10~12모델 스태킹 시도.
    3. **주의**: `GUIDE.md`에서 이미 실패한 방향(층당가, 반복 PL 등)은 절대 시도 금지.

### 🥈 태스크 2. [정밀화] 구별 가중치 최적화 (District-wise Weighting)
* **목표**: Error Analysis에서 나타난 구별 성능 차이를 반영하여, 각 구(Gu)별로 최적의 모델 조합/가중치를 적용.
* **Claude 구현 가이드라인**:
    1. 현재 Ridge 스태킹 결과물에 대하여, 구별로 오차가 큰 지역(성동, 용산, 강남)과 작은 지역(은평)의 예측 패턴을 분석.
    2. **구별 가중치 적용**: 전체 평균 가중치가 아닌, `Gu` 정보를 조건부로 하여 Ridge 모델의 예측값을 보정하거나, 구별로 별도의 메타 모델을 학습하는 방식 시도.

### 🥇 태스크 3. [최종 방어] 고가 구간 사후 보정 (High-end Post-Correction)
* **목표**: 대형 평수 및 고가 아파트의 과소 예측 문제를 해결하기 위해 예측값의 분포를 상향 조정.
* **Claude 구현 알고리즘 순서**:
    1. **오차 분석 기반 계수 산출**: OOF 결과에서 가격 구간별(예: 10억 이하, 10~20억, 20억 이상) 실제값 대비 예측값의 비율(Ratio)을 계산.
    2. **구간별 보정(Piecewise Correction)**: 최종 예측값 $P$에 대하여, 해당 샘플의 가격 구간에 맞는 보정 계수 $K_{price}$를 곱하여 $P_{final} = P 	imes K_{price}$ 산출.
    3. **검증**: 이 보정이 고가 지역의 RMSE를 실제로 낮추는지, 아니면 중저가 지역의 오차를 키우는지 OOF로 엄격히 검증.

## 4. 차세대 고난도 연구 전략 (Next-Generation Advanced Strategies)

단순한 모델 결합을 넘어, 데이터의 본질적 구조를 공략하는 장기적 연구 과제입니다.

### 🚀 전략 A: [공간-시간 통합 임베딩] Spatial-Temporal Graph Transformer
* **핵심 아이디어**: 아파트를 단순한 '행(Row)'이 아닌, 서로 연결된 **'그래프(Graph) 노드'**로 재정의하여 공간적 맥락을 학습합니다.
* **구현 방향**: 
    * 물리적 거리/단지 유사성을 기반으로 Graph를 구축하고, GNN(Graph Neural Network)을 통해 인접 단지의 가격 흐름을 전파(Message Passing)합니다. 
    * 여기에 Time-series Transformer를 결합하여 시간적 변동성까지 포함된 고차원 임베딩을 생성합니다.
* **기대 효과**: "이 아파트가 주변 시세 흐름 속에서 어떤 위치에 있는가"라는 공간적 맥락을 모델이 이해하게 됩니다.

### 🚀 전략 B: [다중 작업 및 분포 정렬] Multi-Task Distribution Alignment (MTL)
* **핵심 아이디어**: 가격(Target)뿐만 아니라 평당가(Unit Price) 등 관련 변수를 **동시에 학습**하여 모델의 '상식'을 강화합니다.
* **구현 방향**: 
    * Deep Learning 기반의 MTL 구조를 설계하여, 하나의 Backbone이 `Target`, `Unit Price`, `Price Distribution`을 각각 예측하는 Multi-head 구조를 만듭니다.
    * 고가 아파트 과소 예측을 막기 위해, 고가 구간 오차에 더 큰 패널티를 주는 **Custom Weighted Loss Function**을 설계합니다.
* **기대 효과**: 가격과 면적 사이의 수학적 관계를 모델이 구조적으로 이해하게 되며, 고가 구간의 편향을 직접적으로 제어할 수 있습니다.

### 🚀 전략 C: [생성적 도메인 적응] Generative Domain Adaptation
* **핵심 아이디어**: Train 데이터(2024-2025)를 **Test 데이터(2026)의 분포로 변환**하여 학습합니다.
* **구현 방향**: 
    * CycleGAN이나 VAE를 활용하여 Train 데이터의 분포를 2026년의 시장 트렌드로 변환하는 생성 모델을 구축합니다.
    * 변환된 '가짜 2026년 데이터'를 활용하여 학습함으로써 모델이 시간적 단절(Domain Shift)을 극복하도록 유도합니다.
* **기대 효과**: 모델이 2026년의 시장 상황을 '미리 경험'하게 하여 시간적 갭으로 인한 성능 저하를 근본적으로 해결합니다.

## 5. 구현 시 주의 사항 및 에러 가이드 (Claude용 팁)

1. **다양성(Diversity) 측정**: 새로운 모델을 추가할 때는 반드시 기존 모델들과의 **Pearson Correlation**을 확인하십시오. 상관계수가 0.95 이상으로 너무 높다면 다양성 기여가 없을 가능성이 큽니다.
2. **Ridge 스태킹의 안정성**: 모델 수가 늘어날수록 Ridge의 $\alpha$ 파라미터 튜닝이 중요해집니다. 
3. **과적합 경계**: Pseudo Labeling과 스태킹이 결합된 상태이므로, 아주 작은 변화에도 OOF와 Public의 격차가 벌어질 수 있습니다. 항상 **OOT(Out-of-Time) 검증**을 병행하여 모델의 일반화 성능을 체크하십시오.
