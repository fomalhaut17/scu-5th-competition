# 제5회 인공지능 모델링 경진대회 - Gemini CLI 전략 분석 및 추천 로드맵

이 문서는 **Gemini CLI**가 분석한 현재 코드베이스의 강점/한계와 실시간 실증 실험 결과를 바탕으로, 소스 개발 담당인 **Claude**가 즉시 구현하고 실험할 수 있도록 정리한 전략 및 개발 가이드라인입니다.

## 1. 현재 코드베이스 종합 평가

### 🟢 핵심 강점 (Best Practices)
* **단계별 레이어 파이프라인 (L1 ~ L4)**: 피처 확정(L1) ➡️ 모델 선발(L2) ➡️ 하이퍼파라미터 튜닝(L3) ➡️ 블렌딩/스태킹(L4) 구조가 명확히 확립되어 있어 체계적이고 안정적인 성능 누적이 가능합니다.
* **구별 트렌드 보정 (GTR/GTR2)**: Train(2024~2025)과 Test(2026) 사이의 뚜렷한 시간적 단절을 메우기 위해 적용된 구별 트렌드 보정은 Public 점수를 대폭 끌어올린 핵심 치트키입니다.
* **신뢰도 필터링 기반 Pseudo Label v2 (`PL2+STK`)**: 단순 Pseudo Labeling의 고질적 문제인 오차 축적(자격 미달 라벨 강화)을 방어하기 위해 **모델 간 합의도 + Fold 간 예측 변동성**을 바탕으로 상위 50%의 확실한 샘플만 Test 데이터에서 수입하여 학습하는 방식은 로컬 OOF RMSE 최선책(2,226)을 도출한 매우 정교하고 우수한 설계입니다.

### 🔴 결정적 한계 (Target Scale Misalignment)
* **로그 변환 학습의 수학적 모순**: 
  현재 모든 모델링 파이프라인은 타겟 변수에 `np.log1p(Target)`을 적용하여 학습하고 최종 예측 시 `np.expm1`로 환원하고 있습니다.
  * 로그 변환 후 MSE를 최적화하는 모델은 본질적으로 **RMSLE (상대 비율 오차)**를 최소화합니다.
  * 그러나 이 대회의 최종 평가 지표는 **원본 스케일(만원) 기준의 absolute RMSE**입니다.
  * 원본 RMSE 평가는 고가 아파트(강남/성동/용산, 120㎡ 이상 대형 평수)의 예측 오차($\text{error}^2$)에 엄청난 제곱 가중치가 붙습니다. 
  * 로그 스케일 모델은 고가 아파트의 큰 절대 오차(예: 1~2억)를 저가 아파트의 상대 오차(예: 1~2천만 원) 수준으로 평평하게 압축해서 학습하기 때문에, 결과적으로 **고가/대형 아파트를 심각하게 과소 예측(Underprediction)**하게 만들어 전체 RMSE를 왜곡시킵니다.

## 2. 실증 실험 데이터 (Log 스케일 vs 원본 스케일)

가설을 증명하기 위해 실제 5-Fold OOF 교차 검증을 통해 **로그 변환 학습(`log1p`) vs 원본 스케일 직접 학습(`Target`)**의 RMSE 성능을 직접 비교 측정했습니다.

| 모델 및 스케일링 방식 | OOF RMSE (만원) | 08 전략(최선) 대비 개선량 | 비고 |
| :--- | :---: | :---: | :--- |
| **CatBoost (Log 스케일 학습 후 `expm1`)** | 2,263.6 | (기준선) | 현재 튜닝 반영된 수준 |
| **CatBoost (원본 스케일 직접 학습)** | **2,247.9** | **-15.7** | **단독 모델로 로그 대비 대폭 우수** |
| **LightGBM (Log 스케일 학습 후 `expm1`)** | 2,288.9 | (기준선) | |
| **LightGBM (원본 스케일 직접 학습)** | **2,282.8** | **-6.1** | |
| ── **앙상블 및 하이브리드** ── | | | |
| **CatBoost (Log + 원본 50:50 단순 평균)** | **2,244.1** | **-19.5** | 단독 모델보다 월등히 개선 |
| **LightGBM (Log + 원본 50:50 단순 평균)** | **2,268.1** | **-20.8** | |
| **4개 모델 단순 평균 (Equal Weight)** | **2,226.1** | **-37.5** | **복잡한 스태킹 없이 단순 평균으로 최고점 갱신** |

### 💡 핵심 인사이트
1. **원본 스케일 타겟 학습의 압승**: 타겟 변환 없이 직접 `Target` 변수를 예측하도록 학습하는 모델이 RMSE 평가 방식에 훨씬 더 적합합니다.
2. **이종 스케일 모델 간의 상호 보완성**: 
   * **로그 모델**: 다수를 차지하는 중저가 주택 예측의 분산(Variance)을 안정화함.
   * **원본 모델**: 극소수 고가/대형 주택 예측의 절대 편향(Bias) 및 과소 예측을 엄격히 방어함.
   * 두 모델을 합쳤을 때 최고의 효율을 보여줍니다.

## 3. Claude를 위한 단계별 개발 로드맵 (Action Plan)

개발 담당자인 **Claude**가 순서대로 구현하고 실험할 수 있도록 정리한 구체적인 태스크 목록입니다.

### 🏃 태스크 1. [즉시 검증] `24_pseudo_label_v2.py` 퍼블릭 제출
* **목표**: 2,226 OOF 성능을 지닌 Pseudo Label v2 파이프라인의 Public Leaderboard 실제 성능 검증
* **상황**: 로컬 테스트(`strategies/_exceptions/24_pseudo_label_v2.py`)와 캐글 업로드용 통합 스크립트(`kaggle_notebooks/24_pseudo_label_v2.py`)는 이미 작성이 완료되어 있으나, 아직 캐글에 제출되지 않은 상태입니다.
* **Claude 수행 작업**: 
  1. `kaggle_notebooks/24_pseudo_label_v2.py` 내용을 캐글 노트북 셀에 복사해 넣습니다.
  2. 세이브 후 제출하여 Public Leaderboard 점수(현재 최선 2,155 돌파 여부)를 기록합니다.

### 🥈 태스크 2. [신규 개발] 하이브리드 타겟 스케일 블렌딩 (Scale-Blending)
* **목표**: 로그 스케일 모델과 원본 스케일 모델의 예측을 결합하여 고가 아파트 과소예측 문제를 정교하게 해결
* **Claude 구현 가이드라인**:
  1. `strategies/L4_blending/02_scale_blending.py` 생성 (또는 `L4_blending`에 신규 추가).
  2. 다음 **4가지 베이스 모델**의 5-Fold OOF 및 Test 예측을 생성합니다:
     * $M_{1}$: `CatBoost` trained on `np.log1p(Target)` ➡️ 예측 결과 환원: `np.expm1(pred)`
     * $M_{2}$: `CatBoost` trained on `Target` directly ➡️ 예측 결과 그대로 사용
     * $M_{3}$: `LightGBM` trained on `np.log1p(Target)` ➡️ 예측 결과 환원: `np.expm1(pred)`
     * $M_{4}$: `LightGBM` trained on `Target` directly ➡️ 예측 결과 그대로 사용
  3. **결합(Ensemble) 방식 실험**:
     * **방법 A (Simple Average)**: 4개 예측값의 산술 평균 ($[M_1 + M_2 + M_3 + M_4] / 4$)
     * **방법 B (Weighted Average)**: 원본 스케일 모델에 좀 더 비중을 둔 가중 평균 (실험 추천 비중: $0.1 \cdot M_1 + 0.3 \cdot M_2 + 0.1 \cdot M_3 + 0.5 \cdot M_4$)
     * **방법 C (Ridge Stacking)**: 4개 OOF 예측값을 독립변수(Feature)로 사용하고, 원본 `Target`을 종속변수로 하여 `Ridge` 회귀 학습 후 메타 메타모델 생성.
  4. 도출된 결합 예측값에 **구별 트렌드 보정(GTR)**을 씌워 최적의 `submission_scale_blend.csv` 도출.

### 🥇 태스크 3. [최종 결합] Scale-Blending + Pseudo Labeling v2 통합 파이프라인
* **목표**: 2026년 데이터 시점 한계 극복(PL2)과 편향 완화(Scale-Blending)를 통합하는 최고 난이도의 최종 후보 모델 구축
* **Claude 구현 알고리즘 순서**:
  1. **[1단계: 의사 라벨링 대상 선별]**
     * 기존 Train 원본 데이터로 4개 모델($M_1, M_2, M_3, M_4$)을 학습합니다.
     * 4개 모델의 예측 평균으로 Test 데이터의 가예측값($P_{\text{test}}$)을 생성합니다.
     * 모델 간 예측 분산과 Fold 간 예측 변동성을 계산하여 가장 예측의 신뢰도가 높은 Test 샘플 상위 50%를 필터링합니다.
  2. **[2단계: 데이터 증강 및 학습 데이터 확장]**
     * 선별된 확실한 Test 샘플에 가예측값($P_{\text{test}}$)을 Target으로 지정(Pseudo Label)하여 Train 원본 데이터와 이어 붙여 확장 데이터셋($D_{\text{aug}}$)을 구성합니다.
  3. **[3단계: 하이브리드 모델 학습]**
     * 확장된 데이터셋($D_{\text{aug}}$)을 사용하여 다시 4개 모델($M_1^*, M_2^*, M_3^*, M_4^*$)을 처음부터 5-Fold 교차 검증으로 학습시킵니다.
  4. **[4단계: 메타 앙상블 및 최적 보정]**
     * 재학습된 4개 모델 예측값을 Ridge 스태킹 또는 50:50 / 가중 평균으로 결합합니다.
     * (주의: OOF 평가는 가짜 데이터가 섞이지 않은 **원본 Train 영역만 추출**하여 RMSE를 계산해야 누수가 없습니다.)
     * 최종 도출된 결합 예측값에 **구별 트렌드 보정(GTR)**을 완벽히 적용하여 `submission_ultimate.csv`를 생성 및 제출합니다.

## 4. 구현 시 주의 사항 및 에러 가이드 (Claude용 팁)

1. **LightGBM의 범주형 가중치**:
   * 원본 스케일 직접 학습 시, LightGBM의 범주형 변수 처리(`as_category=True`)가 원활하지 않으면 특정 카테고리(Gu, Dong)의 가중치가 왜곡되어 RMSE가 튀는 현상이 간혹 발생합니다.
   * 만약 LightGBM 원본 스케일 학습 중 폭발(Explode) 현상이 있다면, 범주형 변수를 원-핫 인코딩하거나 CatBoost처럼 순치(Label Encode) 방식으로 적용해 보십시오.
2. **트렌드 보정 순서 준수**:
   * 모델 학습 단계와 Pseudo Label 생성 단계에서는 원본 스케일 및 로그 스케일 간의 결합에 고도로 연산 집중해야 합니다.
   * `trend_correction` 곱하기는 **모델 예측 결합 및 스태킹이 전부 끝난 가장 최종 결과값**에 한 번만 적용해야 복리 계산 오류가 생기지 않습니다.
3. **누수(Leakage) 엄격 금지**:
   * Pseudo Labeling 적용 및 OOF 측정 시, 테스트 셋의 Pseudo-label이 Fold 교차검증 내에서 학습 셋과 검증 셋으로 스며들지 않도록 독립적으로 관리해야 신뢰할 수 있는 OOF 점수를 얻을 수 있습니다.
