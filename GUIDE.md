## 대회 개요

- **주제**: 서울시 아파트 실거래가 예측 (회귀)
- **평가**: RMSE (낮을수록 좋음)
- **기간**: 2026-06-22 ~ 2026-07-08, 하루 5회 제출
- **플랫폼**: Kaggle (scu-5th-ai-competition)
- **데이터**: Train 1,969건 / Test 531건
- **핵심 특성**: Train은 2024~2025년, Test는 2026년 1~3월 → 시간 갭 존재

## 현재 최선 전략

**전략 28: PL2+UNIT** (Public RMSE **2,096.8**, 1위, 2026-06-25)

```
FE → 4모델 Stage1 → PL 신뢰도 필터(50%) → 데이터 증강
→ 기존 4모델(CB/LGB × log/raw) + 평당가 4모델(Target/Area) = 8모델
→ Ridge(α=10) 스태킹 → 구별 트렌드 보정
```

- Kaggle 노트북: `kaggle_notebooks/28_pl2_unit_price.py`
- OOF RMSE: 2,196
- 핵심: PL2 데이터 증강 + 평당가 크로스 블렌딩(다양성 확보)

## 검증된 교훈

### 트렌드 보정 (실험 완료, 더 이상 시도 불필요)

- **구별 단순평균이 최적** — 동별, EWM, 중앙값, Cap, 외부 데이터 모두 악화
- 보정 강도에 매우 민감: 조금만 과하면 RMSE 1,000 이상 악화
- 부동산 트렌드 자체가 예측하기 어려움 (노이즈 >> 시그널)

### 모델 구조

- 트리 모델 다양성(XGB 추가)은 효과 없음
- Ridge 스태킹은 단순 가중평균 대비 확실히 효과 있음
- 5-Fold가 최적 (10-Fold, 시드 앙상블 효과 없음)
- 분산보다 편향이 주요 문제

### 모델 개선 실험 (2026-06-24, 전부 실패)

4가지 방향을 시도했으나 모두 전략 08보다 악화:
- **메타 모델 복잡화** (CatBoost 메타): 원본 피처 중요도 < 1%, 과적합
- **피처 엔지니어링** (상대 피처): 트리가 이미 인터랙션을 자동 학습하므로 수동 피처 불필요
- **Loss 함수 변경** (Huber): CatBoost에서 delta 설정이 민감, 폭발
- **Residual Modeling**: 같은 피처로는 잔차 학습 불가, 오차 패턴이 유사
- **결론: 전략 08이 현재 피처/데이터 조합에서의 강한 최적점**
- **참고: OOF 개선이 Public 개선을 보장하지 않음** (OOF 13 개선 시 Public 382 개선된 사례)

### 외부 데이터 (실험 완료)

- 공공데이터포털에서 8개 구 실거래가 수집 (2025.10~2026.03)
- 외부 데이터의 구별 트렌드와 대회 데이터의 트렌드가 다름
- 원인: 외부 데이터는 모든 아파트, 대회 데이터는 특정 샘플
- 결론: 외부 트렌드를 직접 보정에 사용 불가

### OOT 검증

- `utils.py`의 `oot_split()`: 마지막 3개월을 holdout하여 시간 기반 검증
- OOT에서는 보정 없음이 최선이었지만, Public에서는 보정이 효과 있음
- OOT와 실제 리더보드의 패턴이 다를 수 있음에 유의

## 다음 작업 방향

### 효과 있었던 방향 (계속 발전시킬 것)
- **PL2 (Pseudo Label v2)**: 신뢰도 필터링 → 데이터 증강 → Public 검증됨
- **이종 스케일 블렌딩**: log/raw + 평당가 등 다양한 타겟으로 다양성 확보
- **크로스 블렌딩**: 서로 다른 관점의 모델을 많이 만들어서 Ridge로 조합

### 시도 가능한 방향
1. **트렌드 보정 α=0.8~0.9**: α=1.2 과보정 확인, α<1.0 쪽 탐색 필요
2. **구별 부분 보정**: 약점 구(강남/성동/용산)만 Ridge 가중치 보정 (전체 교체 아님)
3. **2단 스태킹**: 8모델→Ridge/EN/LGB 3개→최종 블렌드 (depth 확장)
4. **28+32 submission 블렌딩**: 70:30 등 비율로 제출 파일 직접 섞기
5. **PL 임계값 40/60%**: 소폭 탐색

### 핵심 교훈
- OOF와 Public의 갭이 크므로(전략 28: OOF 2,196 → Public 2,096) OOF만으로 판단하면 안됨
- OOF 개선 ≠ Public 개선: 멀티시드(OOF -9 → Public +10), α=1.2(OOF 동일 → Public +14)
- 모델 다양성(크로스 블렌딩)이 개별 모델 성능보다 중요하지만, 유효한 다양성은 평당가뿐
- PL2는 transductive bias 우려에도 실제로 효과 있음 (26 vs 27 비교에서 ~3점 기여 확인)
- 1,969건에서 데이터 분리는 손해, 전체 학습 + Gu 범주형이 최적
- seed=42가 이 테스트셋에 특히 잘 맞음
