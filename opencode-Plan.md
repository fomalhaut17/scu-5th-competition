> GUIDE.md, 결과: 전략 28 Public 2,096.8 1위, 전략 32 OOF 2,187.
> 전략 29~31(타겟변환/반복PL/피처변형) 모두 실패 확인.

## 1. 현황 요약 (v1 대비 변경점)

| 항목 | v1 당시 | 현재 |
|------|---------|------|
| 최선 Public | 2,155 (전략 08) | **2,096.8 (전략 28, 1위)** |
| 최선 OOF | 2,215 (전략 26) | **2,187 (전략 32, 4시드)** |
| 순위 | 3위 | **1위** |
| 잔여 기간 | 13일 | **13일 (∼7/8)** |

### 핵심 발견 (v1→v2)

1. **PL2 효과 입증**: Public 2,096.8 = 전략08(2,155) 대비 **-58점 개선**. transductive leakage 우려에도 실제로 강력하게 작동.
2. **OOF-Public 갭 100점**: OOF 2,187 vs Public 2,096 = ~91점 차이. OOF가 Public을 크게 과소평가. 보정과 PL 모두 Public에서 OOF보다 훨씬 큰 효과.
3. **평당가가 유일한 다양성 축**: 타겟변환/피처변형/반복PL 모두 추가 효과 없음. Target/Area 변환이 유일하게 유효한 이종 스케일.
4. **시드 앙상블 유효**: 4시드 평균 OOF 2,187로 -9점 개선 (기존 "시드 앙상블 효과 없음" 판단 수정 필요).

## 2. 신규 전략 제안

### 전략 A: PL 임계값 최적화 (30%~70% 스윕) ⭐⭐

**근거**: 전략 28의 50% 임계값은 전략 24(4모델)에서 결정된 값. 8모델+Ridge 구조에서는 최적점이 다를 가능성 높음.

```
30% → 1,969건*0.7 + 531*0.3 = ~1,528건 추가 (가장 보수적)
...
70% → 1,969 + 531*0.7 = ~2,341건 추가 (가장 공격적)
```

**방법**: 전략 28 구조에 `threshold_pct` 파라미터 추가 → 30/40/50/60/70% sweep.
단순히 OOF만 비교하지 말고, **구별 OOF 분산 + OOT 점수**도 함께 기록.

**기대**: PL 추가량에 따른 편향-분산 트레이드오프 최적점 발견.

**난이도**: ★☆☆☆☆ (파라미터 1개 변경)

---

### 전략 B: 트렌드 보정 강도 최적화 (구별 계수 + 단리/복리) ⭐⭐⭐

**근거**: 보정이 Public에서 ~100점 기여. 현재 (1+g)^m 복리 보정은 `g`를 구별 월평균 성장률로 고정. 보정 강도가 너무 강하거나 약한 구가 있을 수 있음.

**세부 실험**:

1. **단리 보정**: `1 + g * m` (복리보다 약한 보정, 과보정 위험 감소)
2. **보정 계수 스케일링**: `(1 + alpha * g)^m` 에서 `alpha`를 0.5~1.5 범위 최적화
3. **구별 차등 보정**: 8개 구 각각의 보정 강도를 별도 최적화 (OOF 기반 또는 OOT 기반)

**리스크**: GUIDE.md에서 "보정 강도에 매우 민감" 확인됨. 검증은 OOT + K-fold cross-validation 모두 필요.

**난이도**: ★★☆☆☆

---

### 전략 C: 멀티시드 Ridge 메타 앙상블 ⭐⭐⭐

**근거**: 전략 32는 4개 시드 예측을 단순 평균. Ridge 스태킹이 단순평균보다 항상 우수했던 패턴을 감안하면, 시드 간 다양성을 Ridge가 활용할 가능성 높음.

**방법**:
```
4 seeds × 8 models = 32개 모델 OOF + Test 예측 수집
→ 32개를 입력으로 2차 Ridge 스태킹 (alpha sweep)
→ 단순 평균(전략 32)과 비교
```

**장점**: 각 시드가 서로 다른 PL 샘플 선택, 다른 fold split → 자연스러운 다양성.
구현도 간단 (전략 32의 `tpred_base`, `tpred_unit` 수집만 추가).

**참고**: 전략 29~31에서 20모델 Ridge가 8모델보다 나빴던 이유는 **유의미한 다양성이 없어서**. 시드 변경은 진정한 다양성을 제공하므로 다를 수 있음.

**난이도**: ★☆☆☆☆ (전략 32에 메타 스태킹만 추가)

---

### 전략 D: OOF + OOT 이중 검증으로 최선 선택 ⭐⭐

**근거**: OOF-Public 갭 100점은 OOF만으로 전략 선택이 suboptimal임을 의미. 시간 기반 OOT(Out-of-Time) 검증을 추가해 두 검증 점수의 가중합으로 선택.

**방법**:
1. `utils.py`의 `oot_split()` 사용: 마지막 3개월(2025/10~12) holdout
2. K-fold OOF 점수 + OOT 점수를 각각 계산
3. `final_score = w * OOF_RMSE + (1-w) * OOT_RMSE` (w=0.3~0.7 sweep)
4. OOT이 Public을 더 잘 반영하는지 확인

**용도**: PL 임계값, 보정 강도, Ridge alpha 등 **하이퍼파라미터 선택 기준**으로 활용.

**난이도**: ★★☆☆☆

---

### 전략 E: PL 신뢰도 지표 개선 ⭐⭐

**근거**: 현재 `confidence = 1 - (model_disagreement + fold_cv) / 2`. 4개 모델의 단순 CV 기반. 더 정교한 신뢰도 측정이 PL 품질 향상 가능.

**대안**:
1. **OOF RMSE 가중**: 각 모델의 OOF RMSE로 가중치 부여 → `confidence = weighted combination`
2. **구별 신뢰도**: Gu별로 별도 신뢰도 계산 → 구별로 다른 PL 임계값 적용
3. **잔차 기반**: Stage 1 모델의 OOF 잔차가 작은 샘플 높은 가중치

**난이도**: ★★☆☆☆

---

### 전략 F: 보정 계수 사전 smoothing (구별 성장률 regularization) ⭐

**근거**: 구별 월성장률 `g`는 8개 구 × 평균 2년 = 24개 데이터 포인트로 추정. 노이즈가 큼.

**방법**: Bayesian shrinkage: `g_smoothed = (n*g + lambda*global_mean) / (n + lambda)`
- `n` = 해당 구의 월별 데이터 수
- `lambda` = shrinkage 강도 (5~20 sweep)
- `global_mean` = 전체 8개 구 평균 성장률

**난이도**: ★☆☆☆☆

## 5. 핵심 조언

1. **1위 방어가 최우선**: 하루 5회 중 2회는 전략 28(seed=42)과 전략 32(4시드)로 방어 제출.
2. **OOF 맹신 금지**: OOF-Public 갭 100점. OOF 2,187 개선이 Public 2,096의 개선을 보장하지 않음. OOT 검증을 추가 판단 기준으로 활용할 것.
3. **단순한 변경부터**: 전략 C(메타 스태킹)가 구현 난이도 대비 기대효용 가장 높음.
4. **PL 리스크 인지**: PL2가 transductive leakage에도 효과가 있었지만, 너무 많은 PL 변형(임계값/신뢰도)은 과적합 위험.
5. **잔여 13일 전략**: 1일차: C 구현+테스트, 2일차: A+B 테스트, 이후 최선 조합 제출 반복.
